Maven多模块项目编译失败：程序包xxx不存在

2022-09-11 自己遇到的错误，解决办法

java: 程序包com.heima.utils.common不存在

将xxx.iml 文件删除，然后maven clean，我在terminal哪里mvn clean 一下， 发现了下面的错误

遇到了没又发现这个application运行， 我就试了一下网上说的 使用clean文件，发现就找到这个问题。

大概意思就是说我的子类找不到父类，或者少了pom 打包方式

后面去找xml文件， 发现我的parent哪里写错了。改了回来，再继续试一下。

我就纳闷了，我自己早早就把这个maven3.6.3删了，安装了3.6.1。我后面就去找了config的setting.xml文件，发现自己的localRepository写错了，我就这个改了。

后面就出现了

Could not find goal '' in plugin org.apache.maven.plugins:maven-idea-plugin:2.2.1 among availab

发现是我在terminal哪里 输入 写错了。 下面是正确版本

mvn idea:idea install

如果出现了build的失败的。就把maven源改成了阿里云的源。在config/setting.xml

<mirror>  
     <id>alimaven</id>  
     <name>aliyun maven</name>  
     <url>http://maven.aliyun.com/nexus/content/groups/public/</url>  
     <mirrorOf>central</mirrorOf>          
</mirror>

这样就创建成功了。
